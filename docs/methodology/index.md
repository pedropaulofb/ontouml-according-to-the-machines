# Methodology

This section documents the working methodology used to generate, review, and refine the content of **OntoUML According to the Machines**.

The methodology is concerned with the documentation content itself: how source material is selected, how pages are populated, how generated claims are handled, and how review work is organized. It does not govern the site's visual design, layout, deployment configuration, or general website formatting.

## Purpose

The purpose of the methodology section is to make the project's content-production process explicit and repeatable.

It supports three goals:

1. **Traceability** — record how content is produced and which sources informed it.
2. **Reviewability** — make generated content suitable for later systematic checking and refinement.
3. **Controlled iteration** — separate broad first-pass population from later refinement work.

## Methodological scope

The methodology covers:

- the organization of the project into content-production phases;
- the expected role of selected source material;
- the use of ChatGPT or other generation tools;
- requirements for citations, consulted sources, and generation logs;
- the boundaries between provisional content and reviewed content.

The methodology does not cover:

- visual identity;
- page styling;
- MkDocs theme configuration;
- navigation design beyond content organization needs;
- deployment or infrastructure;
- repository automation, except where automation directly affects content production.

## Relationship to the generation policy

The [Generation Policy](../generation-policy.md) defines the general principles for LLM-generated content in this project.

This methodology section operationalizes those principles by documenting concrete phases of work. Each phase defines its own purpose, execution model, expected outputs, limitations, and completion criteria.

## Project phases

The project is organized into phases. A phase is a content-production period with a defined purpose, scope, and maturity target.

Current phase documentation is available in the [Project Phases](phases/index.md) section.

## General content principles

All phases should preserve the following principles unless a phase-specific document states otherwise:

1. **Generated content remains provisional until reviewed.**
2. **Important claims should be grounded in reported sources.**
3. **Documentation should preserve enough process information to support later review.**
4. **Uncertainty should not be hidden.**
5. **Content work should not be mixed with website styling or visual-design work.**

## Current status

The methodology is expected to evolve with the project. Phase-specific documents may be added or revised as the content-production workflow becomes more mature.
