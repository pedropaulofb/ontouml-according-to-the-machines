# Generation Policy

This project documents OntoUML stereotypes using content generated by large language models.

## Purpose

The purpose of this policy is to make explicit how documentation in this repository is generated, reviewed, and maintained.

## Principles

1. The documentation is generated by large language models.
2. Each stereotype page should preserve a clear record of generation and review activities.
3. Generated content should be checked against authoritative OntoUML sources whenever accuracy matters.
4. The project is unofficial and should not be cited as the normative source for OntoUML.
5. Claims, examples, and interpretations should remain traceable to consulted sources whenever possible.

## Page structure

Each stereotype page should contain the following sections:

- Description
- Stereotype Profile
- Examples
- References
- Generation and Review Log

## References policy

Each page should distinguish between:

### Direct Citations

Direct citations are exact quoted passages used to support specific claims in the page.

### Consulted Sources

Consulted sources are documents, specifications, articles, books, or other materials studied by the LLM while generating or reviewing the page.

A source may appear in the consulted sources list even when no direct quotation from it is used.

## Generation and review log policy

Each stereotype page should include a log entry whenever the page is created, modified, reviewed, corrected, or regenerated.

Each log entry should record:

- date;
- agent name and version;
- action performed;
- short summary of the change;
- main inputs used.

## Review status

The presence of generated content does not imply correctness. Pages should be treated as drafts until reviewed against authoritative OntoUML sources.
