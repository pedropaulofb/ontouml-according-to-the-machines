# Mode

## Description

Mode represents types whose instances are modes: intrinsic moment individuals that inhere in a bearer and are existentially dependent on that bearer. The supplied sources consistently distinguish modes from Quality: qualities are intrinsic moments directly associated with measure or value structures, while modes are intrinsic moments not directly related to such structures or not projectable into a value space. Several sources also contrast modes with Relator as relational moments.

Modes can bear moments of their own, including qualities. The sources also identify externally dependent modes as a special case: they inhere in one entity while depending on another individual independent of their bearer, supporting the representation of relationally grounded intrinsic properties.

In OntoUML profile interpretation, Mode supports the explicit representation of mode universals when the moment itself needs identity, properties of its own, or relations to other entities. The supplied sources associate such explicit mode universals with Characterization relations to the endurant universal they characterize; one source states that every Mode must be directly or indirectly connected to an association end of at least one Characterization relation. This characterization-based connection can also support classifying substantials according to the kind of mode they bear.

The intermediate sources are version-sensitive. Earlier material presents Mode as a stereotype for mode universals in the OntoUML profile, while later OntoUML 2 material treats Mode as identifying the mode nature of instances at the ultimate-sortal level. One transitional source records a proposed separation between the nature of instances and modal properties for mode types; the later source in the set describes Mode as an ultimate-sortal stereotype for classes whose instances are modes. The multilevel source further treats Mode Universal as disjoint from Quality Universal and Relator Universal and states that a second-order type specializing Mode Universal must categorize a Mode Universal base type.
