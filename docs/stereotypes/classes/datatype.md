# Datatype

## Description

Datatype represents quality-structure or value-space content. It is characterized as a semantic reference structure and, in a property-change representation, as a simple way to represent quality structures. When associated with an attribute, a datatype can be understood as the codomain of the attribute function represented by that attribute.

Datatype is distinguished from individuals and from multiply instantiated universals. Its members or instances are values without identity: pure values that may be multiply referred to, rather than ordinary instances of a universal. Quality individuals, by contrast, are bona fide individuals that inhere in bearers. This supports using Datatype for qualia, value regions, codomains, or quality-structure content rather than for the quality individuals themselves.

For complex quality domains, the sources support structured datatypes. A structured Datatype may represent a multidimensional quality structure by using fields for constituent or integral quality dimensions. Constraints and constructors for such structured datatypes should account for the relevant quality dimensions: values should be provided for all integral dimensions, and constraints should reflect the geometry, topology, or ordering structure of the underlying quality structure, including constraints within a dimension and among multiple integral dimensions.
