# Relator

## Source-Specific Description Contribution

The source characterizes relators as entities that are multiply existentially dependent on the related participants. In the discussed OntoUML-style model, the relator is used to account for a relational context that grounds role classification.

This contribution is relevant to the source's treatment of roles: a role is characterized by relational properties, and the relator provides the entity through which the relevant relational dependence is represented. The source uses this mechanism to show that powertype instances, when treated as endurants, can participate in relational structures and bear role classifications.

For Phase 1 purposes, the source supports describing relator as the dependent relational entity that connects participants and grounds role-bearing. It also supports the claim that relators are compatible with the variable-embodiment account because powertype instances can participate in such relational structures without being reduced to abstract universals or mereological sums.
