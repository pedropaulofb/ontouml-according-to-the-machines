# Mixin

## Source-Specific Description Contribution

The source uses Mixin Universal, also called Non-Sortal Universal, for universals abstracting properties common to instances of different sortals. Unlike sortals, mixin universals do not carry a uniform principle of identity; if a mixin universal specialized a sortal, it would inherit a principle of identity, contradicting the definition of mixin universal (Sections 2.1-2.2, pp. 3-4).

The source distinguishes rigid mixins from anti-rigid mixins. Category is the rigid mixin leaf category, while Role Mixin and Phase Mixin are anti-rigid mixin leaf categories. This distinction is used to derive constraints for first-order specialization and for second-order types specializing mixin categories (Section 2.1, p. 3; Section 5.2, p. 11).

For second-order modeling, any specialization of Mixin Universal must have a mixin universal as base type, not a sortal or moment universal. The source further specializes this rule for Category, Phase Mixin, and Role Mixin (Section 5.2, p. 11).
