# Role

## Source-Specific Description Contribution

The source characterizes Role as an anti-rigid sortal whose instances classify substantials through relational properties borne in a relational context. Role differs from Phase because Role classification depends on relational rather than intrinsic criteria (Section 2.1, p. 3).

A Role carries but does not supply a principle of identity and therefore must specialize a unique Kind. The source also treats Role as externally dependent, which follows from its dependence on a relational context (Sections 2.1-2.2, pp. 3-4).

For second-order modeling, a second-order type specializing Role may have any Substantial Universal as base type: Kind, Subkind, Phase, Role, Category, Phase Mixin, or Role Mixin. Such a second-order type categorizes a base type using relational properties as classification criteria. If the base type is a mixin universal, each instance must also specialize a sortal universal so that its principle of identity is settled (Section 5.1.4, pp. 10-11; Section 5.4, p. 18).
