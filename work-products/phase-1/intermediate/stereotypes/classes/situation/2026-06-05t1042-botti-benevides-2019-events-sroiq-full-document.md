# Situation

## Source-Specific Description Contribution

The source uses `Situation` as a UFO-B category that connects endurants, facts, time points, and events. In the formalization, `Situation` is a `ConcreteIndividual`, is disjoint with `Endurant` and `Event`, and is the superclass of `Fact` (p. 15-16, axioms a23-a24, a38-a39).

Situations trigger events and are brought about by events. Each event has exactly one triggering situation and exactly one brought-about situation (p. 10, Section 3.6, S3-S4). When a situation triggers an event, that situation obtains at the event's begin point; when an event brings about a situation, that situation obtains at the event's end point (p. 10, Section 3.6, S1'-S2'). This makes situations the source-supported pre- and post-event correlates used to describe world changes.

The source defines `Fact` as a situation that obtains at a time point (p. 10, Section 3.6, S5). It also uses situations in causal explanation: an event directly causes another event by means of a situation that the first event brings about and that triggers the second event (p. 10, Section 3.6, S6).
