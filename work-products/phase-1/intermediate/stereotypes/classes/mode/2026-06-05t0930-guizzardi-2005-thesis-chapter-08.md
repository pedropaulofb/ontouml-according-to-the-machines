# Mode

## Source-Specific Description Contribution

According to section 8.2.2 and table 8-2, `mode` represents a mode universal, i.e., an intrinsic moment universal. Every instance of a mode universal is existentially dependent on exactly one entity.

The source introduces `mode` because the original UML metamodel lacks constructs for mode universals. In the proposed profile, mode universals are represented by classes stereotyped as `mode`, and the dependence between a mode universal and the endurant universal it characterizes is represented by a `characterization` relation. Every `mode` must be directly or indirectly connected to an association end of at least one `characterization` relation.
