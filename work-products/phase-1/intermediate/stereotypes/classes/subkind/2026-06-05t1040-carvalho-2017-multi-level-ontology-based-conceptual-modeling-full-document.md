# Subkind

## Source-Specific Description Contribution

The source treats Subkind as a rigid sortal specialization of a Kind. A Subkind inherits the identity principle supplied by the Kind rather than supplying its own. Together with Phase and Role, Subkind is characterized as a sortal that must specialize a unique Kind to obtain its principle of identity (Sections 2.1-2.2, pp. 3-4).

For second-order modeling, specializations of Subkind are constrained by rigidity: their instances cannot specialize anti-rigid universals. The detailed discussion states that a second-order type specializing Subkind may partition a Kind, Subkind, or Category using immutable intrinsic classification criteria, while the summary table later marks Kind and Category as admissible base types. This discrepancy should be reviewed in a later phase rather than normalized here (Sections 5.1.2 and 5.4, pp. 8-9, 18).

When the base type is a Category, the source requires each instance of the second-order Subkind type to specialize some Kind, because the base type does not determine the identity principle carried by those instances (Section 5.1.2, p. 9).
