# Quality

## Source-Specific Description Contribution

The source treats `Quality` as a subtype of `Trope`, disjoint from `Disposition`, with both `Quality` and `Disposition` inhering in `ConcreteIndividual` (p. 5, Figure 1; p. 11, Section 3.8; p. 15-17, axioms a16-a18, a37, a77, a101). It explicitly allows events to bear tropes and discusses event qualities as immutable because events themselves are immutable (p. 8, D5; p. 11, Section 3.8).

Qualities are used as reified property instances rather than as literal attributes. A `Quality` instantiates a `QualityUniversal`, and a given `ConcreteIndividual` can bear at most one `Quality` instantiating the same `QualityUniversal` (p. 11, Section 3.8, Q1). A quality has a `qualeOf` relation to a `Quale`, where quales are members of quality structures associated with quality universals (p. 11-12, Section 3.8, Q2-Q3).

The source states that UFO-B does not use datatype properties for these aspects; instead, it handles them through `Quality`, `Quale`, `QualityUniversal`, and `QualityStructure` (p. 32, Section 7.2). This supports a source-specific contribution in which qualities mediate qualitative predication through ontological entities rather than through datatype-valued attributes.
