# Category

## Source-Specific Description Contribution

The source characterizes Category as a rigid mixin universal that represents properties applying necessarily to instances of different kinds. It is rigid but non-sortal; therefore it abstracts over multiple identity principles rather than supplying a single one (Section 2.1, p. 3).

Because Category is rigid, it cannot specialize anti-rigid universals. Because it is a mixin universal, it also cannot have a sortal as base type in second-order patterns. The source therefore restricts a second-order type specializing Category to a Category base type (Section 5.2.1, p. 11; Section 5.4, p. 18).

The source describes Category specialization patterns as hierarchies of categories in which more specific categories may partition a more general category according to immutable intrinsic properties. This supports using Category for rigid, non-sortal taxonomic abstractions over entities of multiple kinds (Section 5.2.1, pp. 11-12).
