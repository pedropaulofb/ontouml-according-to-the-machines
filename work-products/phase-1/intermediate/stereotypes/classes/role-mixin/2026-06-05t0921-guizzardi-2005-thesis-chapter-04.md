# Role Mixin

## Source-Specific Description Contribution

A `role-mixin` represents an anti-rigid non-sortal that abstracts common contingent properties of roles. The source characterizes role mixins as dependent anti-rigid non-sortals and treats them as more general than formal roles.

Because a `role-mixin` is a non-sortal, it does not supply a principle of identity. Because it is anti-rigid, it represents contingent classification rather than essential classification. The source situates `roleMixin` as the appropriate abstraction when common role properties apply across multiple disjoint role types.

The proposed profile requires `roleMixin` to be defined as an abstract class. It also constrains its supertype so that it is not a `kind`, `subkind`, `phase`, or `role`. When an association represents the restriction condition of a `roleMixin`, the minimum cardinality at that association end must be at least one.
