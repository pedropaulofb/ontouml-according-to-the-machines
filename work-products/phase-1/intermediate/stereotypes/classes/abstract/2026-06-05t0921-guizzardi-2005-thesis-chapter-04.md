# Abstract

## Source-Specific Description Contribution

In this source, abstractness is used as a UML class property required by parts of the proposed profile, rather than as one of the leaf ontological categories in the typology of substantial universals. The source characterizes an abstract class as a class that cannot have direct instances.

The source uses abstractness as a modeling constraint in two places relevant to later stereotype consolidation. First, when phases form a phase-partition of a substance sortal, the restricted superclass must be abstract because the phase partition is complete and disjoint in the mathematical sense. Second, mixins must be depicted as abstract classes because they are non-sortals and cannot have direct instances.
