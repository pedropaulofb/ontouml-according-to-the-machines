# Type

## Source-Specific Description Contribution

The source treats powertypes as types used when a domain's classification scheme is itself part of the subject matter. A powertype is analyzed as a type whose instances combine type-like and instance-like characteristics: they are associated with lower-level types but are also treated as entities about which properties, modal classifications, and change can be represented.

In the UML powertype pattern discussed by the source, a powertype is connected to a base type through a generalization-set configuration. Its instances correspond to explicit subtypes of the base type, while base-type instances are connected to the corresponding powertype instances through an `isClassifiedBy` relation. The paper stresses that this modeling pattern raises an ontological question concerning the status of powertype instances.

The source considers two interpretations insufficient for the modeling requirements it identifies. Interpreting powertype instances as abstract universals does not support qualitative change, contingent existence, accidental properties, or cross-world variation. Interpreting them as mereological sums or populations gives them extensional identity and makes their identity depend directly on membership, which conflicts with the intended stability of the modeled entity.

The source's preferred contribution is to interpret powertype instances as variable embodiments. On this account, a powertype instance is an endurant whose manifestation may vary by world or time according to an identity principle associated with a corresponding lower-level type. This interpretation is used to support direct properties, regularity properties, resultant properties, qualitative change, contingent modal classification, and the possibility of being unmanifested while still referable.
