# Phase

## Source-Specific Description Contribution

The source characterizes Phase as an anti-rigid sortal whose instances classify substantials according to intrinsic properties. Phase differs from Role because Phase classification is based on intrinsic properties rather than relational properties (Section 2.1, p. 3).

For first-order specialization, Phase carries but does not supply an identity principle; therefore, a Phase must specialize a unique Kind that supplies that principle. Because Phase is anti-rigid, it may specialize rigid universals without contradiction, but it remains constrained as a sortal (Section 2.2, p. 4).

For second-order modeling, a second-order type specializing Phase may have any Substantial Universal as base type: Kind, Subkind, Phase, Role, Category, Phase Mixin, or Role Mixin. The source presents the typical cross-level relation as partitioning, using mutable intrinsic properties as classification criteria. If the base type is a mixin universal, each instance must still specialize a kind to settle its principle of identity (Section 5.1.3, p. 10; Section 5.4, p. 18).
