# Creation

## Source-Specific Description Contribution

The source introduces `<<creation>>` as a special kind of participation. It relates an event to an endurant that is created in that event.

The source treats `<<creation>>` as part of the event-to-endurant connection introduced by the event extension. Since the modeled event is a past occurrence, the created endurant associated with the event is part of the fixed history represented by the model.

As with participation, the source's immutability rule applies to association ends attached to endurants in creation associations: they should be read-only because a past event cannot change which endurant it created.
