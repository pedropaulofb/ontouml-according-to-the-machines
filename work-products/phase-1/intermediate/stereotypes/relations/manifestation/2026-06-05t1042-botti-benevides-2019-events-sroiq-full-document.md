# Manifestation

## Source-Specific Description Contribution

The source explains atomic events as manifestations of dispositions. In the FOL formalization, each `AtomicEvent` is manifested by a unique `Disposition`, using `manifestedBy` from `Disposition` to `AtomicEvent` and treating manifestation as the inverse reading (p. 6-7, Section 3.3, D2; p. 17, axiom a118).

Manifestation connects events to their bearers through dispositions. If a disposition inheres in an object and is manifested by an event, that event depends on the object (p. 6-7, Section 3.3, D4; p. 19, axiom a135). A situation triggers an atomic event exactly when a disposition is activated by that situation and manifested by the event in the FOL theory (p. 7, Section 3.3, D3).

The source further constrains the temporal-situational context of manifestation: if a disposition is manifested by an event, the disposition is present in the situation that triggers the event and in the situation the event brings about (p. 8, Section 3.3, D8; p. 19, axioms a137-a138).
