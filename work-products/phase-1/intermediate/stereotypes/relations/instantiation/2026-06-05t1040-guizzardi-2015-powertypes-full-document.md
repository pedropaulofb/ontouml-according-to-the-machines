# Instantiation

## Source-Specific Description Contribution

The source treats instantiation as the ordinary classification relation between an individual and a type, and it examines how this relation behaves in multi-level modeling. It notes that some multi-level approaches treat links between adjacent classification levels uniformly, thereby allowing entities to appear both as instances and as types.

In the rejected higher-order-universal interpretation, a powertype instance would itself be a universal and would instantiate a higher-order universal. The source rejects this as insufficient for the intended modal behavior of powertype instances, because abstract universals cannot change, contingently exist, or differ across possible worlds in the required way.

In the preferred account, instantiation remains central but is coordinated with the `isClassifiedBy` relation. For a base type and a powertype, each subtype of the base type is associated with a unique powertype instance; an ordinary individual instantiates that subtype exactly when it is classified by the associated powertype instance. Instantiation also supplies the principle used to determine the membership of the rigid embodiments that manifest variable embodiments.
